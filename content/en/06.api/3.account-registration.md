# Authentication Register

## Endpoint Overview

| Property          | Value                |
| ----------------- | -------------------- |
| **Method**        | `POST`               |
| **URL**           | `/api/auth/register` |
| **Content-Type**  | `application/json`   |
| **Accept**        | `application/json`   |
| **Auth Required** | No (Public)          |

---

## Request Headers

| Header         | Value              |
| -------------- | ------------------ |
| `Content-Type` | `application/json` |
| `Accept`       | `application/json` |

---

## Request Parameters

| Parameter         | Type     | Required   | Description                                                                                           |
| ----------------- | -------- | ---------- | ----------------------------------------------------------------------------------------------------- |
| `email`           | `string` | ✅ Required | The email address for the new account. Must be a valid, unique email.                                 |
| `password`        | `string` | ✅ Required | The account password. Must meet complexity requirements.                                              |
| `username`        | `string` | ✅ Required | The display name for the new user account.                                                            |
| `turnstile_token` | `string` | ⬜ Optional | Cloudflare Turnstile CAPTCHA token. Pass an empty string if CAPTCHA is not enabled.                   |
| `invite_code`     | `string` | ⬜ Optional | Referral/invite code provided by an existing user. Required when invite-only registration is enabled. |

---

## Example Request Body

```json
{
  "email": "admin@gmail.com",
  "password": "123456789aA",
  "username": "Admin",
  "turnstile_token": "",
  "invite_code": ""
}
```

---

## Response

### `200 OK` — Success

| Field     | Type     | Description                                              |
| --------- | -------- | -------------------------------------------------------- |
| `message` | `string` | Human-readable confirmation that registration succeeded. |

### Example Response Body

```json
{
  "message": "Registration successful"
}
```

---

## Code Sample — PHP (cURL)

```php
<?php
$curl = curl_init();

curl_setopt_array($curl, [
  CURLOPT_URL            => 'http://127.5.5.5/api/auth/register',
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_FOLLOWLOCATION => true,
  CURLOPT_HTTP_VERSION   => CURL_HTTP_VERSION_1_1,
  CURLOPT_CUSTOMREQUEST  => 'POST',
  CURLOPT_POSTFIELDS     => json_encode([
    'email'           => 'admin@gmail.com',
    'password'        => '123456789aA',
    'username'        => 'Admin',
    'turnstile_token' => '',
    'invite_code'     => '',
  ]),
  CURLOPT_HTTPHEADER => [
    'Content-Type: application/json',
  ],
]);

$response = curl_exec($curl);
curl_close($curl);

$data = json_decode($response, true);
echo $data['message']; // "Registration successful"
```

## Common Error Responses

| Status Code | Meaning               | Likely Cause                                                          |
| ----------- | --------------------- | --------------------------------------------------------------------- |
| `409`       | Conflict              | The email or username is already associated with an existing account. |
| `422`       | Unprocessable Entity  | A required field is missing or fails validation (e.g. weak password). |
| `403`       | Forbidden             | Registration is currently disabled (`registration_status: false`).    |
| `429`       | Too Many Requests     | Too many registration attempts from the same IP in a short period.    |
| `500`       | Internal Server Error | An unexpected server-side error occurred.                             |
