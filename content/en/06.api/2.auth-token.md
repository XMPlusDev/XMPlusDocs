# Authentication Login

## Endpoint Overview

| Property          | Value              |
| ----------------- | ------------------ |
| **Method**        | `POST`             |
| **URL**           | `/api/auth/login`  |
| **Content-Type**  | `application/json` |
| **Accept**        | `application/json` |
| **Auth Required** | No (Public)        |

---

## Request Headers

| Header         | Value              |
| -------------- | ------------------ |
| `Content-Type` | `application/json` |
| `Accept`       | `application/json` |

---

## Request Body Parameters

| Parameter         | Type      | Required   | Description                                                                    |
| ----------------- | --------- | ---------- | ------------------------------------------------------------------------------ |
| `email`           | `string`  | ✅ Required | The user's registered email address.                                           |
| `password`        | `string`  | ✅ Required | The user's account password.                                                   |
| `remember`        | `boolean` | ⬜ Optional | If `true`, the token will have an extended expiry. Defaults to `false`.        |
| `type`            | `string`  | ✅ Required | Authentication provider type. Use `"local"` for standard email/password login. |
| `turnstile_token` | `string`  | ⬜ Optional | Cloudflare Turnstile CAPTCHA token. Pass an empty string if not required.      |

### Example Request Body

```json
{
  "email": "admin@gmail.com",
  "password": "123456789aA",
  "remember": true,
  "type": "local", # telegram, google, github
  "turnstile_token": "" # require if captcha enabled
}
```

---

## Response

### `200 OK` — Success

| Field        | Type                | Description                                                                                      |
| ------------ | ------------------- | ------------------------------------------------------------------------------------------------ |
| `message`    | `string`            | Human-readable status message confirming the login outcome.                                      |
| `ip`         | `string`            | The IP address from which the login request originated.                                          |
| `token`      | `string`            | Bearer token for authenticating subsequent API requests. Use as `Authorization: Bearer {token}`. |
| `userid`     | `integer`           | Unique numeric identifier of the authenticated user.                                             |
| `username`   | `string`            | Display username of the authenticated user.                                                      |
| `expires_at` | `string (ISO 8601)` | Token expiry date/time. Format: `YYYY-MM-DDTHH:MM:SS+00:00`.                                     |

### Example Response Body

```json
{
  "message": "Sign in successful",
  "ip": "127.0.0.1",
  "token": "132|wFEgnE4E87dF3Jr6BvnrhnfRb2K2GLz9FqWX691ia05f1cf0",
  "userid": 1,
  "username": "Admin",
  "expires_at": "2026-01-26T09:24:11+00:00"
}
```

## Code Sample — PHP (cURL)

```php
<?php
$curl = curl_init();

curl_setopt_array($curl, [
  CURLOPT_URL            => 'http://127.5.5.5/api/auth/login',
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_FOLLOWLOCATION => true,
  CURLOPT_HTTP_VERSION   => CURL_HTTP_VERSION_1_1,
  CURLOPT_CUSTOMREQUEST  => 'POST',
  CURLOPT_POSTFIELDS     => json_encode([
    'email'           => 'admin@gmail.com',
    'password'        => '123456789aA',
    'remember'        => true,
    'type'            => 'local',
    'turnstile_token' => '',
  ]),
  CURLOPT_HTTPHEADER => [
    'Accept: application/json',
    'Content-Type: application/json',
  ],
]);

$response = curl_exec($curl);
curl_close($curl);

$data = json_decode($response, true);
echo $data['token']; // Use token for future requests
```

## Common Error Responses

| Status Code | Meaning               | Likely Cause                                         |
| ----------- | --------------------- | ---------------------------------------------------- |
| `401`       | Unauthorized          | Email or password is incorrect.                      |
| `422`       | Unprocessable Entity  | A required field is missing or improperly formatted. |
| `429`       | Too Many Requests     | Too many login attempts in a short period.           |
| `500`       | Internal Server Error | An unexpected server-side error occurred.            |
